Sorting Algorithm Mechanism Best Case Complexity Average Case Complexity Worst Case Complexity After 1st Pass Behavior Working
Bubble Sort Comparison / Greedy O(n) O(n^2) O(n^2) Largest to Rightmost Compares adjacent elements and swaps them if they are in the wrong order. After each pass, the largest unsorted element moves to the rightmost position.
Selection Sort Selection and Sorting O(n^2) O(n^2) O(n^2) Smallest to Leftmost Divides the input list into two parts: sorted and unsorted. Repeatedly selects the smallest element from the unsorted part and moves it to the sorted part.
Insertion Sort Comparison O(n) O(n^2) O(n^2) Partially Sorted Builds the sorted array one element at a time. Compares each element with the already sorted part and inserts it in its correct position.
Merge Sort Divide and Conquer O(n log n) O(n log n) O(n log n) Fully Sorted Divides the array into halves recursively until single elements are reached. Merges the sorted halves to create a fully sorted array.
Quick Sort Divide and Conquer, Partitioning O(n log n) O(n log n) O(n^2) Partially Sorted Selects a pivot element, partitions the array into elements smaller and larger than the pivot, and recursively sorts the subarrays.
Heap Sort Binary Heap O(n log n) O(n log n) O(n log n) Partially Sorted (Max Heap) Builds a max-heap (or min-heap) from the array. Repeatedly extracts the root element (max or min) and swaps it with the last element, then heapifies the reduced heap.
